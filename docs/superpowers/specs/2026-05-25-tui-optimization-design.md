# wallpaper-magpie TUI 优化设计方案

**日期**: 2026-05-25

## 项目背景

wallpaper-magpie 是一个基于 Rust + ratatui 的 TUI 壁纸下载工具，支持三个壁纸源：Unsplash、Pexels、Wallhaven。当前 TUI 向导有三个页面：源选择、筛选条件配置、确认下载。

## 目标

通过模块化重构，优化 TUI 交互体验，修复已知 Bug，增强 CLI 帮助功能。

## 设计方案

### 一、页面 1：壁纸源选择

#### 布局
```
┌─ 壁纸源 ──────────────────────┐
│ > ● Wallhaven                │ ← 默认选中，绿色，一直可用
│   ○ Unsplash                 │ ← 灰色（未配置 API Key）
│   ○ Pexels                   │ ← 灰色（未配置 API Key）
│                               │
│ [Enter] 下一步               │
│ [↑/↓] 切换  [e] 编辑 API Key │
└───────────────────────────────┘
```

#### 改动要点
1. 标题从"可选来源"改为"壁纸源"
2. 源顺序：Wallhaven 排第一，默认选中（索引 0）
3. 可用性指示：
   - **绿色** = 可用（Wallhaven 始终可用）
   - **灰色** = 需配置 API Key（Unsplash、Pexels 未配置时）
   - 使用 `style.fg()` 渲染颜色
4. **e 键**：弹出编辑窗口（模态框），可输入/修改当前选中源的 API Key
5. 模态框设计：
   - 标题"编辑 API Key"
   - 单行输入框
   - Enter 保存，Esc 取消
   - 保存后写入项目目录内的 `config.toml`
   - 立即更新颜色状态

### 二、页面 2：筛选条件配置

#### 布局
```
┌─ 筛选条件 ──────────────────────────────┐
│ 关键词 (q / query)      [________]     │
│ 分类 (categories)       [风景 ▼]       │ ← 枚举类型，显示下拉菜单
│ 纯度 (purity)           [SFW ▼]        │
│ 排序 (sorting)          [日期 ▼]       │
│ 分辨率 (resolutions)    [________]     │ ← 输入框，带占位符提示
│ 数量 (per_page)         [10____]       │
│ ...                                     │
│ [Enter] 下一步 (非编辑状态)             │
│ [↑/↓] 切换 [e] 编辑值 [Esc] 上一步     │
└─────────────────────────────────────────┘
```

#### 改动要点
1. **每个源独立配置**：根据 `docs/unsplash.md`, `docs/pexels.md`, `docs/wallhaven.md` 分别定义参数
2. **显示格式**：`中文名 (参数名)`，如 `关键词 (q)`
3. **枚举类型**（如 orientation, purity, sorting）：弹出下拉菜单（Pop-up Dropdown）
   - 上下选择，Enter 确认，Esc 关闭
4. **输入框类型**：
   - 数字型：占位符提示范围，如 `1-30`
   - 字符串型：占位符提示格式，如 `1920x1080`
   - 留空 = 不使用该条件
5. **默认值**：每个参数预设合理默认值
6. **编辑模式**：按 `e` 进入当前参数的编辑模式
   - Enter 确认编辑
   - Esc 取消编辑
7. **非编辑状态下按 Enter**：进入下一步（跳到确认页）
8. **Bug 修复**：修复左侧箭头指示器只有"下一步"显示的问题，所有可选项都应显示 `>` 箭头

### 三、页面 3：下载进度

#### 布局
```
┌─ 下载进度 ───────────────────────────────┐
│ 总数: 24  已完成: 12  进行中: 3           │
│                                         │
│ 日志:                                   │
│ ▶ 开始下载: wallpaper_001.jpg           │
│ ✓ 完成: wallpaper_001.jpg               │
│ ✗ 失败: wallpaper_002.jpg (timeout)     │
│ ▶ 开始下载: wallpaper_003.jpg           │
│ ... (自动滚动)                          │
│                                         │
│ [Enter] 完成  [Esc] 取消 (需确认)        │
└──────────────────────────────────────────┘
```

#### 改动要点
1. 显示统计：总数、已完成数、进行中数
2. 日志窗口：显示所有操作记录（开始/成功/失败），带自动滚动
3. **Esc 取消机制**：
   - 第一次按 Esc：弹出确认对话框"确定要取消吗？已下载的文件将保留"
   - 第二次按 Esc：确认取消，返回上一页
   - 取消后不删除已下载文件
4. **Enter**：下载完成后按 Enter 退出

### 四、键位与底部帮助栏

#### 全局底部帮助栏（每个页面都显示）
```
[↑/↓] 切换  [Enter] 确认/下一步  [Esc] 退出/上一步  [e] 编辑
```

#### 各页面具体键位

| 页面 | 键位 | 功能 |
|------|------|------|
| 页面1 | `e` | 编辑当前选中源的 API Key |
| 页面1 | `Enter` | 确认选择，进入页面2 |
| 页面1 | `Esc` | 取消 API Key 编辑 / 退出程序 |
| 页面2（非编辑） | `↑/↓` | 切换筛选条件 |
| 页面2（编辑中） | 输入字符 | 修改值 |
| 页面2（枚举） | `Enter` | 打开下拉菜单 |
| 页面2 | `e` | 进入编辑模式 |
| 页面2（非编辑） | `Enter` | 进入下一步（页面3） |
| 页面2 | `Esc` | 取消参数编辑 / 返回页面1 |
| 页面3 | `Enter` | 确认完成 |
| 页面3 | `Esc` | 弹出取消确认，再按一次确认 |

#### 键位变更
- **移除 `q` 键**：不再绑定退出功能

### 五、CLI 帮助增强

#### 新增子命令
```bash
wallpaper-magpie help unsplash    # 显示 Unsplash 筛选参数说明
wallpaper-magpie help pexels      # 显示 Pexels 筛选参数说明
wallpaper-magpie help wallhaven   # 显示 Wallhaven 筛选参数说明
```

#### 输出格式示例
```
Unsplash 筛选参数:
  query        搜索关键词
  orientation  方向 (landscape|portrait|square)
  ...
```

### 六、技术实现要点

#### 配置文件
- 路径：项目目录内的 `config.toml`
- 存储各源的 API Key 配置
- 不涉及工作目录以外的文件（如 `~/.config/`）

#### 数据驱动筛选条件
- 每个源独立定义筛选条件配置
- 支持类型：字符串、数字、枚举
- 包含：中文名、参数名、类型、默认值、占位符提示

#### 模态框组件
- 复用的弹窗组件
- 支持标题、内容、确认/取消按钮
- 用于：API Key 编辑、枚举下拉菜单、取消确认对话框

## 验收标准

1. 页面1：Wallhaven 排在第一且默认选中，可用源绿色，不可用源灰色
2. 页面1：按 `e` 可编辑 API Key，保存后立即生效
3. 页面2：显示各源独立筛选条件，显示 `中文名 (参数名)` 格式
4. 页面2：枚举类型使用下拉菜单选择，输入框有占位符提示
5. 页面2：非编辑状态 Enter 进入下一步
6. 页面3：显示下载进度和日志，Esc 两次确认取消
7. 所有页面：底部显示当前页面键位帮助
8. CLI：`help <source>` 显示对应源的筛选参数
9. Bug 修复：页面2所有可选项都显示左侧箭头指示器
10. `q` 键不再绑定退出功能

## 附录：源筛选条件定义

### Wallhaven 参数
- `q` - 搜索关键词 (字符串，留空不限制)
- `categories` - 分类 (枚举: 111/110/101/011/100/010/001)
- `purity` - 纯度 (枚举: SFW/Sketchy/NSFW)
- `sorting` - 排序 (枚举: date_added/relevance/random/views/favorites/toplist)
- `order` - 顺序 (枚举: desc/asc)
- `topRange` - 时间范围 (枚举: 1d/3d/1w/1M/3M/6M/1y)
- `atleast` - 最小分辨率 (字符串，如 1920x1080)
- `resolutions` - 精确分辨率 (字符串)
- `ratios` - 宽高比 (字符串)
- `colors` - 颜色 (字符串)
- `page` - 页码 (数字)
- `per_page` - 每页数量 (数字，默认 24)

### Unsplash 参数
- `query` - 搜索关键词 (字符串)
- `page` - 页码 (数字)
- `per_page` - 每页数量 (数字，默认 10)
- `order_by` - 排序 (枚举: relevant/latest)
- `orientation` - 方向 (枚举: landscape/portrait/squarish)
- `color` - 颜色 (字符串)

### Pexels 参数
- `query` - 搜索关键词 (字符串)
- `page` - 页码 (数字)
- `per_page` - 每页数量 (数字，默认 10)
- `orientation` - 方向 (枚举: landscape/portrait/square)
- `size` - 尺寸 (枚举: large/medium/small)
- `color` - 颜色 (字符串)
